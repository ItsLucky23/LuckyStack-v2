## Headline

The CLI **core** (`packages/cli/src`) is genuinely hardened — all 3 wave-2 runs agree and current source confirms: npm hijack closed, JSON.parse guarded, install ordering fixed, O(n²) scan fixed, multiline env parsing fixed, `--ci` divergence removed, `checkI18n` decomposed. The unanimous 6/6 wave-1 build-breaker (`ActiveSession` token/handle drift, #1) is **FIXED**.

All residual defects cluster in the shipped **login asset bundle** (`assets/login/**`) and its `create-luckystack-app/template` twin. Two items deserve attention:
- **CLI-01 (HIGH, still OPEN, no longer "now-fixed"):** a credentials user can NEVER self-delete via the shipped UI — the server requires a password whenever `dbUser.password` is set, but `settings/page.tsx:284` sends only `{ confirmation: 'DELETE' }`, no password. Guaranteed dead button (GDPR right-to-erasure break). Run-1 wave-2 marked #1 "now-fixed" referring to the *type-drift* #1, NOT this UI/server password mismatch — they are different #1s. Verified OPEN.
- **CLI-W1 / wave-1 #5 (MEDIUM, OPEN):** the inverse edge — a null/empty-hash credentials account skips the password gate and is deletable with `DELETE` alone (`deleteAccount_v1.ts:29` gates on hash truthiness, not provider).

No new criticals. The two security-framed NEW items from wave-2 (CLI-W4 language poisoning, the template-updateUser "arbitrary theme/language" claim) were verified DOWN to low/false: theme/language are DB enums (run-4 skeptic) and the language regex only affects the attacker's own session.

## Feature ideas (forwarded, not findings)

- `luckystack verify-assets` + single-source asset/template generation — kills the entire asset↔template drift class (O4, F1, D1, O11) rather than policing it after the fact (`ASSET_AHEAD_OF_TEMPLATE` is a permanent escape hatch).
- Post-inject `tsc --noEmit` gate in `add login`/`add presence` — F1 shipped for six audit rounds precisely because nothing compiled the injected output. A `--check` step turns "asset is broken" into a clean `add` failure.
- Per-package `envKeys` manifest feeding `check-env` — removes the hand-maintained `FRAMEWORK_ENV_KEYS` drift (O12).
