## Headline — the wave-2 CRITICAL is REAL and ship-blocking (verified)

The shipped live renderer reads the WRONG artifact shape. Confirmed end-to-end in current source:

- **Emitter output is FLAT arrays.** `packages/devkit/src/typeMap/emitterArtifacts.ts:62` declares `apis: Record<string, ApiDocsEntry[]>`; `:269` `docsData.apis[pagePath] = []`; `:281` `.push({ page, name, version, ... })`. Top-level `syncs` is also a flat array per page (`:63`, `:423`, `:435`).
- **Committed artifact confirms it.** `src/docs/apiDocs.generated.json:3` → `"playground": [ { "page": "playground", "name": "echo", ... }, ... ]` (array), plus a top-level `"syncs": { "playground": [ ... ] }` (line 269).
- **Live renderer assumes a NESTED map.** `packages/docs-ui/src/docsHtml.ts:339-342` iterates `Object.entries(apis)` → `Object.entries(names)` → `Object.entries(versions)` (page→name→version→meta). Against `apis.playground = [ {entry}, ... ]`, `Object.entries(array)` yields `["0", {entry}]`, then `Object.entries({entry})` iterates the entry's OWN fields, so `name="0"`, `version="page"`, `meta="playground"` (a string). Every real route renders as scrambled nonsense rows; `meta.method` on a string is `undefined` → falls back to POST.
- **Syncs are never rendered.** `render()` (`docsHtml.ts:385`) reads only `data.apis`; no `data.syncs` reference anywhere in the shipped script.
- **The correct renderer is DEAD code.** `renderCore.ts` (`renderDocsEndpoint`/`renderDocsSyncEntry`, flat-array + sync-aware + i18n) is imported ONLY by `renderCore.test.ts` (grep: no `renderCore` import in `index.ts`/`docsHtml.ts`/`register.ts`; no `.toString()` call on any renderCore export). `tsup.config.ts` entries are `index.ts` + `register.ts` only, so renderCore ships to no entry.
- **Bug is masked by a wrong-shaped test.** `liveRenderCore.test.ts:156-164` feeds the SHIPPED `buildGroups` a hand-built NESTED fixture (`playground: { echo: { v1: {...} } }`) that matches the buggy traversal, not the real array — so the 59-test suite is green over the broken path.
- **Dist ships the bug.** `packages/docs-ui/dist/chunk-XARUZSGO.js` contains the same nested `Object.entries` iteration and `meta.tags && meta.tags.length` guard — published 0.2.0 is affected.

This is NEW (no wave-1 run saw it — the merged baseline says "one of the cleaner, best-tested packages … no Critical or High except the prod-gate"). The wave-1 LOW_ANALYSIS even documents the live path as "the nested map" and explicitly left the renderCore→live rewire DEFERRED as out-of-scope. Severity: **CRITICAL** (package's sole function broken against every real artifact). Blast-radius temper: dev-only, defaults `enabledInProd:false`, so impact is broken/garbled dev docs + a false sense of a delivered fix, not an end-user/security breach — but it is ship-blocking for the 0.2.0 publish goal.
