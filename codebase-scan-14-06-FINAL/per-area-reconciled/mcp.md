# Reconciled area report: mcp

Branch `chore/package-split-prep`, HEAD 302cbf1 + ~248 uncommitted (this week's fixes). Verified against the CURRENT working tree (every verdict below opened the cited file).

Scope: `packages/mcp/src/{index.ts, artifacts.ts}` (+ `artifacts.test.ts`, `tsup.config.ts`, `package.json`, and the producer `scripts/generateGraph.mjs`). A small, read-only stdio MCP server exposing committed AI-context artifacts (dependency graph, decisions, routes, runbooks, capabilities) as ~8 Claude Code tools.

## Headline

The package was **heavily hardened this session** (wave-1 LOW_ANALYSIS "FIXED-NOW" + "Safe-sweep" passes). All wave-1 HIGH/MEDIUM functional bugs are FIXED in the current tree and independently re-verified here. **Zero critical/high security findings remain** — unanimous across all 3 wave-2 runs and confirmed: no network, no writes, no SQL, zod-validated at the SDK boundary, a path-containment guard now present.

The one genuinely **NEW** correctness defect wave-2 surfaced (all 3 runs, high confidence) is **MCP-01: `JSON.parse` in `loadGraph` is outside any try/catch** — it falsifies the wave-1 "handler catch is unreachable" WONT-FIX (A16) for the single most likely corruption mode (a torn/merge-conflicted `docs/ai-graph.json`). Everything else open is low/medium quality-of-degradation polish. Single-file build is intentional (main = bin = `dist/index.js`).

Wave-2 corroboration counts are over the 3 real runs (run-1 mcp.md, run-3 mcp.md, run-4 small-pkgs.md mcp slice). Wave-1 counts are over the 6-run merge (n/6).
