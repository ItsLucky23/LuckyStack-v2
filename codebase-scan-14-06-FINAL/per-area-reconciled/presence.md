## Headline

The wave-1 high-severity cluster (unguarded async / unhandled rejections in the grace timer, connect path, intentional-disconnect listener; the `clientSwitchedTab` leak; sampler false-negative; silent swallow; magic sentinels; duplication) is **genuinely fixed in the current tree** — verified each one at file:line, every fix carries an explanatory `//?` comment. No CRITICAL/HIGH defect survives inside the package.

The **one substantive risk that survives** is the room-name formatter asymmetry (wave-1 #2): the JOIN sites across the whole monorepo (`@luckystack/server` `loadSocket.ts:226,414,418`, `@luckystack/sync` `handleSyncRequest.ts:722` / `handleHttpSyncRequest.ts:619`) join/broadcast on RAW room codes, while presence's broadcast + snapshot route through `formatRoomName`. A non-identity `registerRoomNameFormatter` (the documented multi-tenant primitive) silently breaks ALL presence fan-out + snapshots. The IN-package half is now internally consistent (both `informRoomPeers`/`informRoomPeersLeft` AND `getRoomPresence` apply the formatter — the snapshot half was fixed this session), so the remaining fix is cross-package (server/sync join sites). Tracked OPEN-known / cross-package DEFERRED.

Wave-2 also surfaced a real cluster wave-1 under-rated: **per-socket vs per-token AFK** (cross-instance activity Map → remote peers always `afk:false`; multi-tab user double-fires / false-AFK). These are NEW (medium/low), real, and tied to the same module-level `lastActivityBySocket` keyed by socket.id.
