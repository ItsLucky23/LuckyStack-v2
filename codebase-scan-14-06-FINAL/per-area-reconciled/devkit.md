## Headline

The devkit area is substantially hardened. The two most-corroborated wave-1 HIGHs are FIXED and re-verified: root-sync route-key divergence (#2 — loader/routeMeta/templateInjector all return `'system'`) and the supervisor missing-`'error'`-listener crash (#3). The supervisor was rewritten (the 30s blocking `runPreRestartHook`/`spawnSync`, #9, is gone; a once-guard dedupes error/exit). Codegen hardened: `validateGeneratedTypeIdentifiers` re-promoted to `throw` (#12/#25), `splitTopLevel` quote-aware + angle-clamped (#14), `extractClientInputFromFile` scoped + null-on-unbalanced (#21), the DELETE-twin splice uses replacer functions (#4 partial), comment-only files templated only on `add` (#5), `calculateRelativePath` srcDir-anchored (#11).

Two real themes survive. (1) **The template-injector's AST migration is only half-done** — `updateClientFileForPairedServer` (paired twin) and `injectServerTemplateWithClientInput` are still regex/string source-mutation that corrupt nested-brace types and `$`-sequences. (2) **A NEW, wave-1-missed `src/` hardcode** (DEVKIT-01/-02) that silently drops every route on a non-`src` `srcDir` layout — the highest-value new finding, because it directly defeats the advertised `projectConfig.paths.srcDir` config key.

One wave-2 NEW finding is a **FALSE POSITIVE**: DK-07 (validateDeploy ANSI codes "missing ESC byte") — the ESC byte (0x1b) IS present in source (`od -c` shows octal `033` before every `[31m`/`[0m`); the Read tool just doesn't render it.
