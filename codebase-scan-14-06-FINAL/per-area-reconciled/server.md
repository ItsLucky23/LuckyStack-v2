Reconciles the 3 wave-2 server.md runs (run-1, run-3, run-4; run-2 was the input digest, ignored) against the wave-1 6-scan MERGED baseline (`codebase-scan-14-06-MERGED/per-area/server.md`) and the CURRENT working tree (HEAD `302cbf1` + ~248 uncommitted fixes). Every kept finding was re-verified by opening the cited file at the current line; no wave-2 "fixed/present" claim was trusted on faith.

Commit/tree note: the wave-2 runs all claim several wave-1 items are now fixed. I independently confirmed those (resolveRequesterIp helper, logSanitize depth guard, SSE init assertion, withSessionLock defuse, requestId-at-top, HEAD CSRF parity, SERVE_DENYLIST source-disclosure block). They are real fixes and are recorded as FIXED below.

Headline verdict: **no CRITICAL in this package.** Real open items: 1 HIGH bug regression (api-SSE twin drift, SRV-01), 1 HIGH security default (/_health secret fingerprint, #1), plus a cluster of MEDIUM/LOW correctness + drift items. There is NO `LOW_ANALYSIS/server.md` in the wave-1 merge (server was not in that batch), so no prior FIXED/DEFERRED session record exists for this area to cross-check against.
