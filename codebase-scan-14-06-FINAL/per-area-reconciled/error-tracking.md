## Headline

The wave-1 flagship — **ET-02 per-request identity (ALS) never bound in production (their #1, 6/6 HIGH)** — is **genuinely FIXED** for the adapter path. All four request handlers (`handleApiRequest.ts:379,399`, `handleHttpApiRequest.ts:91,162`, `handleSyncRequest.ts:177,326`, `handleHttpSyncRequest.ts:176,213`) open `runWithErrorTrackerIdentityScope(...)` before the first interleaving await and write the user via `setCurrentErrorTrackerIdentity(...)` after `readSession`. All three adapters read `getCurrentErrorTrackerIdentity()` at capture time (PostHog `posthog.ts:66-70`, Datadog `datadog.ts:105-106`, Sentry `sentry.ts:68-72`). Wave-1 #2 (Datadog never reads ALS) also fixed.

The dominant REMAINING theme, corroborated 3/3, is the **"shadow-API" cluster**: typed/documented/tested surfaces never wired into prod — `flushErrorTrackers()` not called on shutdown (data loss), the package barrel omits 5 documented exports, `posthogConfig.ts` is fully dead, `registerPreCaptureFilter` can't see the error. Plus two live secret-leak paths on the **legacy `initializeSentry()` slot** (raw `extra`, and process-global identity bleed).

**No Critical.** Highest live items: ERRORTRACKING-01 (flush-on-shutdown data loss, HIGH), the legacy-Sentry raw-`extra` PII path (HIGH→MED), and the legacy-Sentry process-global identity bleed (HIGH→MED, NEW nuance wave-2 caught that wave-1 missed for the legacy path).
